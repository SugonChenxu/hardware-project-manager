# PRD — 项目计划

> 版本: v1.2 | 日期: 2026-07-02 | 状态: 已实现（含L6/L9子任务拆分）

---

## 1. 模块概述

**模块名称**：项目计划  
**页面路由**：`/plan`  
**目标用户**：硬件项目管理者 X（个人使用）  
**核心定位**：基于里程碑模板一键生成硬件项目排期，支持行内编辑、关键路径识别、甘特图可视化和版本历史回溯。

**设计原则**：
- 一键排期：内置 M1~M5 标准里程碑模板，点击即可以今日为基准生成
- 所见即编辑：日期、周期、任务名均可点击内联编辑，无需弹窗
- 关键路径驱动：标准 CPM 算法自动识别制约整体工期的瓶颈链路

### 3.2 表格视图

**阶段分组行**（背景 `#fafafa`，底部 `2px` 分隔线）：
- 左侧色块 + 阶段名称
- 同行右侧彩色徽标显示：`开始日期 → 结束日期 (N天)`

**任务行**（缩进 24px）：
- 任务名：点击进入编辑，回车/失焦保存，空值不删除
- 开始日期：点击弹出 DatePicker，修改后联动周期和后续节点
- 结束日期：点击弹出 DatePicker，与开始日期 + 周期保持联动
- 周期(天)：点击进入 InputNumber，修改后联动结束日期
- 状态：自动判定，Tag 展示（已完成/进行中/未开始）

### 3.3 行内插入任务

| 项目 | 规格 |
|------|------|
| 触发 | 操作列 `[+]` 按钮（InsertRowBelowOutlined 图标） |
| 行为 | 在当前行下方插入同阶段新任务 |
| 初始值 | 任务名="新任务"，开始日期=上行结束日期，周期=7天 |
| 排序 | 自动重新编号，继承同阶段 parallelGroup |

### 3.4 状态自动判定

| 状态 | 判定规则 | 标签样式 |
|------|---------|---------|
| 已完成 | 结束日期 < 今天 | 灰色 Tag + 行 45% 透明度 |
| 进行中 | 开始日期 ≤ 今天 ≤ 结束日期 | 蓝色 Tag |
| 未开始 | 开始日期 > 今天 | 灰色 Tag |

### 3.5 日期修改级联

| 修改 | 级联行为 |
|------|---------|
| 修改关联任务的日期/周期 | 后续关联任务整体平移（跳过未关联任务块时追踪块内最晚结束日期） |
| 修改独立任务的日期 | 仅自身变化；若延长后超出后续关联任务起点，自动推动后续关联任务 |
| 锁定节点 | lockStart/lockEnd 对应的日期不参与级联偏移 |
| 并行块内任务 | linked=false，互不级联；级联链路自动跳过整块 |

### 3.6 关键路径检测（自动判定，不可手动设置）

算法分三步：

1. **按 phaseGroup 分桶** — 大阶段之间是串行的
2. **桶内扫描时间窗口重叠** — 聚类为「并行子组」（单任务=串行组，不标记）
3. **仅并行子组（≥2个重叠任务）标记瓶颈** — endDate 最晚的节点 = 关键路径，其余 = 并行分支

> 串行链路中的任务不标记为关键路径。仅在出现同时间段内多个并行任务时，自动识别瓶颈并标记 `★ 关键`。

### 3.7 日期锁定（开始/结束独立）

| 项目 | 规格 |
|------|------|
| 触发 | 开始日期/结束日期旁的 🔒/🔓 图标按钮 |
| lockStart | 锁定开始日期，修改开始日期时拒绝变更 |
| lockEnd | 锁定结束日期（即交付节点），结束日期固定不变 |
| 级联保护 | lockStart=true 的后续任务开始日期不变；lockEnd=true 的后续任务结束日期不变、周期自动重算 |
| 甘特图 | 存在任意锁定时显示 🔒 图标 |
| 删除保护 | lockEnd=true 时删除按钮 disabled |

### 3.8 任务关联/独立

| 项目 | 规格 |
|------|------|
| 默认状态 | 所有任务 `linked=true`（后一个任务的开始时间 = 前一个任务的结束时间） |
| 触发 | 操作列 `LinkOutlined` 按钮 |
| 解除关联 | `linked=false`，变为独立节点，日期修改不影响前后任务，也不被前后任务影响 |
| 建立关联 | 自动将开始日期对齐到前置任务结束日期，并重建后续关联链 |
| 视觉效果 | 蓝色图标=已关联，灰色图标=独立 |

### 3.9 甘特图

- 表头：按**月份刻度**显示日期标签
- 条：根据 stage 颜色半透明，关键路径条实色 + 红色边框
- 点击条直接跳转编辑该任务的结束日期
- 底部图例：各阶段色块 + ★关键路径 + 🔒锁定节点

### 3.10 保存与撤销

| 操作 | 行为 |
|------|------|
| 确认保存 | 写入 localStorage，自动生成版本快照 |
| 撤销 | 丢弃未保存修改，从 localStorage 恢复最近一次保存状态 |
| isDirty 标记 | 任何编辑后置为 true，顶部显示黄色 Alert 提示 |
| 仅保存时可用 | 按钮 disabled 直到 isDirty=true |

### 3.11 版本历史

| 项目 | 规格 |
|------|------|
| 触发 | 每次「确认保存」自动创建快照 |
| 查看 | 顶部「版本历史 (N)」按钮 → Modal 弹出 Timeline |
| 恢复 | 点击某版本的「恢复」按钮，当前计划立即替换为该快照 |
| 数据 | 完整的 PlanPhase[] JSON，不包含模板信息 |

### 3.12 导出 Excel（含公式）

| 字段 | 导出方式 |
|------|---------|
| 阶段 / 任务名称 | 存文本值 |
| 开始日期 | 存文本值 |
| 周期(天) | 存数值 |
| 结束日期 | **公式** `=DATEVALUE(Cx)+Dx`，在 Excel 端改开始日期或周期后自动联动 |

这样用户可以在 Excel 中调整排期，再通过「更新模板」上传回系统。

### 3.13 更新模板（Excel 导入）

| 项目 | 规格 |
|------|------|
| 触发 | 「更新模板」按钮 → 选择 .xlsx/.xls 文件 |
| 解析 | 自动识别「阶段/任务/开始/结束/周期」列 |
| 兼容 | 支持 Excel 日期序列号 和 字符串日期格式 |
| 生效 | 导入后覆盖内置模板，下次「创建计划」使用新模板 |

## 4. 数据结构

```typescript
type PhaseStatus = 'completed' | 'in_progress' | 'upcoming';

interface PlanPhase {
  id: string;
  projectId: string;
  phaseGroup: string;      // 阶段分组（M1预研阶段 / M2计划阶段 / ...）
  taskName: string;         // 任务名称
  startDate: string;        // YYYY-MM-DD
  endDate: string;          // YYYY-MM-DD
  duration: number;         // 周期（天）
  lockStart: boolean;       // 开始日期锁定
  lockEnd: boolean;         // 结束日期锁定
  linked: boolean;          // 是否与前后任务串行关联（默认 true）
  isCriticalPath: boolean;  // 是否关键路径
  isParallel: boolean;      // 是否并行分支
  parallelGroup: string;    // 并行组标识
  status: PhaseStatus;      // 运行时状态
  sortOrder: number;        // 排序序号
}

interface PlanTemplatePhase {
  phaseGroup: string;
  taskName: string;
  startDate: string;
  endDate: string;
  duration: number;
  parallelGroup: string;
}

interface PlanTemplate {
  name: string;
  phases: PlanTemplatePhase[];
  importedAt: string;
}

interface PlanHistory {
  id: string;
  projectId: string;
  version: number;
  label: string;
  snapshot: PlanPhase[];
  createdAt: string;
}
```

**存储**：
- `plan_phases` → `PlanPhase[]`（localStorage）
- `plan_template` → `PlanTemplate | null`（localStorage）
- `plan_history` → `PlanHistory[]`（localStorage）
- 通过 `usePlanStore` (Zustand) 管理

## 5. 交互细节

| 场景 | 行为 |
|------|------|
| 未选项目 | 显示"请先在顶部选择项目" |
| 无排期数据 | 显示 Empty + 「创建计划」按钮 |
| 点击开始/结束日期 | DatePicker 展开，选中后自动关闭编辑态 |
| 点击周期 | InputNumber 展开，失焦保存 |
| 点击任务名 | 变为 Input 输入框，回车/失焦保存 |
| 修改日期 | 自动级联后续节点（锁定节点跳过） |
| 修改周期 | 结束日期 = 开始日期 + 新周期 |
| 锁定开始/结束日期 | 日期旁锁定图标切换，禁止修改/删除 |
| 甘特图条点击 | 跳转到该任务的结束日期编辑 |
| 大阶段增删 | 工具栏「添加大阶段」按钮（弹窗输名称），大阶段行操作列编辑/删除 |
| 任务关联切换 | 操作列 LinkOutlined 按钮（蓝色=关联/灰色=独立） |
| 未保存切换项目 | 当前计划保留在内存（localStorage 只写一份） |
| 页面刷新 | 从 localStorage 恢复最近保存的版本 |
