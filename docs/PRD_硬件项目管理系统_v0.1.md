## 一、项目背景

当前硬件项目（服务器/模组开发）使用腾讯文档多维表格管理，存在以下痛点：
- 多项目并行时，进度不直观，缺乏全局视图
- 任务/问题跟踪分散在多个表格，难以汇总
- 物料/BOM 管理依赖人工维护，容易遗漏
- 项目成员/人力拆解缺乏可视化
- 无法自动预警延期风险

目标：构建一个**个人使用的硬件项目管理 Web App**，覆盖从立项到收尾的全流程管理。

开发要求：
1.每次更新代码后，需要在github上同步，并且要在对应PRD章节中输出相应改动
2.建立各模块的PRD，里面要写明每个模块功能的逻辑
3.每个模块都要具有数据保存功能，数据可以存在本地，防止意外情况数据丢失，无论任何情况，项目数据一定要不能丢失！
4.各个模块间相同的功能最好要采用相同的逻辑代码
5.严格做好代码记录
6.每一个大功能模块开发完成后，提醒我进行代码检查。
7.任何模块开发都要注意美观性

### 关键路径检测（自动判定，不可手动设置）

[](https://github.com/SugonChenxu/hardware-project-manager/blob/master/docs/PRD-%E9%A1%B9%E7%9B%AE%E8%AE%A1%E5%88%92.md#36-%E5%85%B3%E9%94%AE%E8%B7%AF%E5%BE%84%E6%A3%80%E6%B5%8B%E8%87%AA%E5%8A%A8%E5%88%A4%E5%AE%9A%E4%B8%8D%E5%8F%AF%E6%89%8B%E5%8A%A8%E8%AE%BE%E7%BD%AE)

算法分三步：

1. **按 phaseGroup 分桶** — 大阶段之间是串行的
2. **桶内扫描时间窗口重叠** — 聚类为「并行子组」（单任务=串行组，不标记）
3. **仅并行子组（≥2个重叠任务）标记瓶颈** — endDate 最晚的节点 = 关键路径，其余 = 并行分支

> 串行链路中的任务不标记为关键路径。仅在出现同时间段内多个并行任务时，自动识别瓶颈并标记 `★ 关键`。

### 3.7 日期锁定（开始/结束独立）

[](https://github.com/SugonChenxu/hardware-project-manager/blob/master/docs/PRD-%E9%A1%B9%E7%9B%AE%E8%AE%A1%E5%88%92.md#37-%E6%97%A5%E6%9C%9F%E9%94%81%E5%AE%9A%E5%BC%80%E5%A7%8B%E7%BB%93%E6%9D%9F%E7%8B%AC%E7%AB%8B)

|项目|规格|
|---|---|
|触发|开始日期/结束日期旁的 🔒/🔓 图标按钮|
|lockStart|锁定开始日期，修改开始日期时拒绝变更|
|lockEnd|锁定结束日期（即交付节点），结束日期固定不变|

### 更新 2：父子任务层级展示

**需求描述**：当前系统没有体现出父任务与子任务的关系。像 L6 和 L9 这种具有子任务的父任务，需要体现出来。

**层级逻辑**（来自用户需求）：
- **大阶段**（M1~M5）：起始日期和结束日期由其下的父任务起止时间决定
- **父任务**（L1~L12）：没有子任务时，起始日期由其自己或前置父任务结束日期决定；有子任务时，由其子任务起止时间自动决定，**不可编辑**
- **子任务**（L1.1、L6-1 等）：开始时间由前置任务结束时间决定
- **孙任务**：一般不会有，若有则逻辑同上

**实现要求**：
- 父任务行要有视觉标识（如加粗、缩进不同）
- 子任务行缩进显示，体现层级关系
- 父任务有子任务时，日期自动计算（取子任务最早开始、最晚结束）
- 父任务无子任务时，可手动编辑日期

**状态**：待实现

---

### 更新 3：日期逻辑修正

**需求描述**：当前日期修改逻辑不符合预期，需要修正。

**正确逻辑**：
1. **修改开始日期**：周期默认不变，结束日期 = 开始日期 + 周期。后续关联任务的开���日期跟随逻辑自动变化。
2. **修改结束日期**：开始日期不变化，周期 = 结束日期 - 开始日期。
3. **修改周期**：开始日期不变化，结束日期 = 开始日期 + 新周期。

**关键原则**：
- 任何日期的改动，后续的关联任务都应该跟随逻辑自动变化
- 模板只是用来节省一个一个模块创建的时间，不要设置成不变化的。**日期应该跟着逻辑走，不应该一成不变**

**实现要求**：
- 修改开始日期 → 结束日期联动，级联后续关联任务
- 修改结束日期 → 周期联动，不级联（因为开始日期不变）
- 修改周期 → 结束日期联动，级联后续关联任务
- 级联更新时要跳过锁定的节点

### 更新 4：任务关联/独立功能澄清

**需求描述**：日期关联功能，是要求关联后，开始日期结束日期会跟随前后任务产生变化。独立的时候，代表该任务与前后不存在依赖关系，所以日期不跟随前后变化。这个在视觉上要和时间锁定区分开。

**关联（linked=true）**：
- 后一个任务的开始日期 = 前一个任务的结束日期
- 前一个任务日期修改时，后一个任务开始日期自动对齐
- 视觉：蓝色链接图标

**独立（linked=false）**：
- 任务与前后不存在依赖关系
- 日期修改不影响前后任务，也不被前后任务影响
- 视觉：灰色断开图标

**与锁定的视觉区分**：
- 关联/独立：用链接图标（🔗/🔓）
- 锁定：用锁定图标（🔒/🔓）
- 两个功能独立，可以组合（如：关联 + 锁定开始日期）

### 更新 5：日期锁定功能强化

**锁定逻辑**：

- **lockEnd=true**：结束日期固定不动，无论如何不可以再更改
- 锁定后的日期，即使级联更新也不变化
- 锁定节点的删除操作应该被禁止（因为有结束日期锁定）

- 鼠标悬停锁定单元格时，显示提示"已锁定，不可修改"

- 锁定时，日期单元格只读（不可点击编辑）
