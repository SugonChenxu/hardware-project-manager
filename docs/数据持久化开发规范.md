# 数据持久化开发规范

## 概述

本文档定义了项目中所有模块的数据持久化实现标准，确保用户数据不会丢失。

---

## 核心原则

1. **自动保存** - 每次数据修改都自动保存到 IndexedDB
2. **加载恢复** - 应用启动时自动从 IndexedDB 恢复数据
3. **降级机制** - IndexedDB 失败时降级到 localStorage
4. **防抖优化** - 避免频繁写入，提升性能

---

## 实现标准

### 1. Store 结构

每个 store 必须包含以下部分：

```typescript
import { create } from 'zustand';
import { dbGet, dbSet } from '../utils/db';

interface MyStore {
  items: MyItem[];
  load: () => Promise<void>;
  add: (data: Omit<MyItem, 'id' | 'createdAt'>) => MyItem;
  update: (id: string, data: Partial<MyItem>) => void;
  remove: (id: string) => void;
}

export const useMyStore = create<MyStore>((set, get) => ({
  items: [],

  // 1. 加载数据
  load: async () => {
    try {
      const items = await dbGet<MyItem[]>('my_items', []);
      set({ items });
    } catch (e) {
      console.error('加载数据失败:', e);
    }
  },

  // 2. 添加数据
  add: (data) => {
    const item: MyItem = {
      ...data,
      id: generateId(),
      createdAt: new Date().toISOString(),
    };
    const items = [item, ...get().items];
    set({ items });
    // 注意：不需要在这里调用 dbSet，subscribe 会自动保存
    return item;
  },

  // 3. 更新数据
  update: (id, data) => {
    const items = get().items.map((i) =>
      i.id === id ? { ...i, ...data } : i
    );
    set({ items });
  },

  // 4. 删除数据
  remove: (id) => {
    const items = get().items.filter((i) => i.id !== id);
    set({ items });
  },
}));

// 5. 自动保存：监听 state 变化
let saveTimeout: ReturnType<typeof setTimeout> | null = null;
useMyStore.subscribe((state) => {
  if (state.items.length > 0) {
    // 防抖：延迟 500ms 保存
    if (saveTimeout) {
      clearTimeout(saveTimeout);
    }
    saveTimeout = setTimeout(() => {
      dbSet('my_items', state.items).then(() => {
        console.log('💾 自动保存完成，共', state.items.length, '条数据');
      }).catch(e => {
        console.error('❌ 自动保存失败:', e);
      });
    }, 500);
  }
});
```

---

## 关键要点

### 1. 加载数据

- Store 必须提供 `load` 方法
- 在组件挂载时调用 `load()`
- 在 `App.tsx` 中统一加载所有 store

```typescript
// 在组件中
useEffect(() => {
  load();
}, []);
```

### 2. 自动保存

- 使用 `subscribe` 监听 state 变化
- 添加 500ms 防抖，避免频繁写入
- 每次变化都保存到 IndexedDB

### 3. 数据迁移

- 从 localStorage 迁移到 IndexedDB
- 添加迁移标记，避免重复迁移
- 在 `load` 方法中调用 `migrateFromLocalStorage()`

```typescript
load: async () => {
  try {
    // 先尝试从 localStorage 迁移
    await migrateFromLocalStorage();
    
    // 从 IndexedDB 加载
    const items = await dbGet<MyItem[]>('my_items', []);
    set({ items });
  } catch (e) {
    console.error('加载数据失败:', e);
  }
},
```

## 工具函数

### `dbGet<T>(key: string, defaultValue: T): Promise<T>`

从 IndexedDB 读取数据，失败时降级到 localStorage。

### `dbSet<T>(key: string, value: T): Promise<void>`

保存到 IndexedDB，同时备份到 localStorage。

### `migrateFromLocalStorage(): Promise<void>`

从 localStorage 迁移数据到 IndexedDB（一次性）。

---

## 检查清单

开发新模块时，确保：

- [ ] Store 包含 `load` 方法
- [ ] 组件挂载时调用 `load()`
- [ ] Store 底部有 `subscribe` 监听器
- [ ] `subscribe` 中有 500ms 防抖
- [ ] `load` 方法中调用 `migrateFromLocalStorage()`
- [ ] 所有 CRUD 操作都更新 state（让 subscribe 自动保存）
- [ ] TypeScript 编译通过

## 常见问题

### Q: 数据没有保存？

**A**: 检查 `subscribe` 是否正确实现，是否有防抖逻辑。

### Q: 刷新后数据消失？

**A**: 检查 `load` 方法是否正确实现，是否在组件挂载时调用。

### Q: 控制台报错？

**A**: 检查 `dbGet` 和 `dbSet` 的导入是否正确，IndexedDB 是否支持。
