# PRD — 会议纪要

> 版本: v1.0 | 日期: 2026-07-02 | 状态: 开发中

---

## 1. 模块概述

**模块名称**：会议纪要  
**页面路由**：`/meeting-notes`  
**目标用户**：硬件项目管理者 X（个人使用）  
**核心定位**：自动管理项目例会记录，集成腾讯会议 AI 纪要，自动生成项目周报。

**设计原则**：
- 卡片式布局：每场会议独立卡片展示
- 自动化：自动导入腾讯会议链接和 AI 纪要
- 智能化：基于 AI 总结自动生成周报

## 3. 功能清单

### 3.1 卡片式布局

| 项目 | 规格 |
|------|------|
| 布局 | 卡片式，每场会议一个卡片 |
| 卡片内容 | 会议日期、腾讯会议链接、AI 记录状态、操作按钮 |
| 卡片操作 | 查看原始记录、查看 AI 总结、生成周报、删除 |
| 排序 | 按会议日期倒序 |

### 3.2 导入腾讯会议链接

| 项目 | 规格 |
|------|------|
| 触发方式 | 点击「导入会议链接」按钮，输入会议 ID 或链接 |
| 自动获取 | 根据会议 ID 自动获取会议详情（标题、时间、参与者） |
| 存储 | 保存到 IndexedDB |
| 状态标记 | 未导入 / 已导入 / AI 记录已生成 |

**腾讯会议 API 集成（模拟）**：
- 由于实际 API 需要认证，先实现模拟数据
- 用户可以手动输入会议 ID，系统模拟获取会议信息
- 后续可接入真实的腾讯会议 API

### 3.3 读取 AI 会议原始记录

| 项目 | 规格 |
|------|------|
| 触发方式 | 点击卡片上的「查看记录」按钮 |
| 数据来源 | 腾讯会议 AI 生成的原始记录（模拟 API 响应） |
| 展示 | Markdown 格式渲染，支持复制 |
| 存储 | 保存到 IndexedDB |

**AI 记录内容（模拟）**：
```markdown
# 项目例会 #1 - 2026-07-02

## 参会人员
- 张三（项目经理）
- 李四（硬件工程师）
- 王五（软件工程师）

## 讨论内容
1. 项目进度汇报
   - M3 研发测试阶段进展顺利
   - PCB 洗板已完成，正在进行 PCBA 打板
2. 问题讨论
   - 散热设计需要优化
   - 建议提前准备 A02 设计
3. 下一步计划
   - 继续 EVT 测试
   - 准备 L9 并行测试
```

### 3.4 同步获取 AI 自动总结

| 项目 | 规格 |
|------|------|
| 触发方式 | 点击卡片上的「查看总结」按钮 |
| 数据来源 | 腾讯会议 AI 自动生成的总结（模拟 API 响应） |
| 展示 | 结构化展示：进展、问题、计划 |
| 存储 | 保存到 IndexedDB |

**AI 总结内容（模拟）**：

### 3.5 自动生成项目周报

| 项目 | 规格 |
|------|------|
| 触发方式 | 点击「生成周报」按钮（卡片级别或全局） |
| 数据来源 | 所有会议的 AI 总结内容 |
| 生成逻辑 | 基于模板自动生成结构化周报 |
| 输出格式 | Markdown / Word / PDF |
| 预览 | 在页面底部预览周报内容 |
| 导出 | 支持导出为 Word/PDF |

**周报模板**：
```markdown
# 项目周报（2026-07-02 ~ 2026-07-08）

## 一、本周进展
- [从 AI 总结中提取]

## 二、下周计划
- [从 AI 总结中提取]

## 三、风险与问题
- [从 AI 总结中提取]

## 四、会议纪要链接
- [会议 1](链接)
- [会议 2](链接)
```

## 4. 数据结构

```typescript
interface MeetingNote {
  id: string;
  projectId: string;
  meetingId: string;          // 腾讯会议 ID
  meetingUrl: string;         // 腾讯会议链接
  title: string;              // 会议标题
  date: string;               // 会议日期 YYYY-MM-DD
  attendees: string[];        // 参会人员
  aiTranscript: string;       // AI 原始记录（Markdown）
  aiSummary: {                // AI 自动总结
    progress: string[];
    issues: string[];
    plans: string[];
  } | null;
  weeklyReport: string;       // 生成的周报内容
  weeklyReportGenerated: boolean;
  createdAt: string;
  updatedAt: string;
}
```

**存储**：
- `meeting_notes` → `MeetingNote[]`（IndexedDB）
- 通过 `useMeetingNoteStore` (Zustand) 管理

## 5. 交互细节

| 场景 | 行为 |
|------|------|
| 点击「导入会议链接」 | 弹出对话框，输入会议 ID 或链接 |
| 点击「查看记录」 | 侧边栏或模态框展示 AI 原始记录 |
| 点击「查看总结」 | 侧边栏或模态框展示 AI 总结 |
| 点击「生成周报」 | 基于 AI 总结生成周报，预览并支持导出 |
| 点击「删除」 | 确认后删除会议记录 |
| 页面加载 | 从 IndexedDB 加载所有会议记录 |
